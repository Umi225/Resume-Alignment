# 需求文档 (Requirements)

> 状态：MVP 规划中  
> 最后更新：2026-05-13  
> 当前项目真实状态：仅完成数据类型层 (`src/types/resume.ts`)

---

## 产品定位

「AI JD 对齐简历工作台」不是"一键生成简历"工具，而是：
- 用户长期沉淀经历资产的素材库
- AI 辅助分析 JD、推荐匹配经历、基于 STAR 法则优化表达
- 用户全程可控、可编辑
- 最终导出高质量 PDF 简历

核心原则：**人机协同、用户可控、AI 不胡编**

---

## MVP 功能范围

### P0 — 必须有（MVP 发布）

#### 1. 经历资产管理
- 增删改查六类经历：教育、工作、项目、获奖、证书、技能
- 经历列表页（卡片视图）
- 经历编辑表单（按类型动态渲染字段）
- 基础标签系统（tags）

#### 2. 简历编辑器
- 左右分屏：左侧编辑面板 + 右侧 A4 实时预览
- 从经历库选择内容加入简历
- 区块排序（拖拽）与显隐控制
- 模板切换（至少 1 套模板）
- 版本保存（支持保存多个版本）

#### 3. PDF 导出
- 浏览器打印方案（`@media print` + `window.print()`）
- A4 尺寸预览
- 中文排版优化

#### 4. 数据持久化
- LocalStorage 兜底（MVP 阶段不接入后端）

---

### P1 — 重要（发布后迭代）

#### 5. JD 对齐工作台
- JD 文本粘贴
- AI 分析 JD 关键词与要求
- 经历匹配推荐（按匹配度排序）
- STAR 法则优化建议（对比视图）
- 对齐后的内容一键进入简历

#### 6. AI 辅助（基础版）
- 经历 bullets 逐条优化
- AI 建议必须显式采纳，不覆盖原文
- 流式输出（打字机效果）

#### 7. 多模板支持
- 至少 2 套模板（校招风格 + 社招风格）
- 模板配置（字体、主题色、间距）

---

### P2 — 优化（远期）

#### 8. 高级 AI 功能
- 自动提取技能标签
- 自动 STAR 结构化
- 简历评分与改进建议

#### 9. 数据同步
- 后端 API + 数据库持久化
- 多设备同步

#### 10. 导入导出
- 从 PDF/Word 解析导入
- JSON 数据备份与恢复

---

## 非功能需求

- **中文互联网校招审美**：黑白灰为主，克制留白，思源黑体/苹方字体栈
- **性能**：经历库 100 条以内无卡顿，预览实时响应
- **浏览器兼容**：Chrome / Edge / Safari 最新两个大版本
- **打印质量**：PDF 导出与屏幕预览像素级一致

---

## 明确不做（当前阶段）

- [ ] 用户登录 / 账号系统
- [ ] 后端服务 / 数据库存储（MVP 用 LocalStorage）
- [ ] 从 PDF/Word 反向解析导入
- [ ] 多人协作
- [ ] 移动端编辑（仅保证预览可用）
